# 개발 로그 (Logs)

프로젝트를 진행하면서 새로 알게 된 내용, 트러블슈팅 경험, 의사결정 기록을 시간순으로 남기는 공간입니다.

## 작성 규칙

- 파일명은 `YYYY-MM-DD-주제.md` 형식으로 작성합니다.
  - 예) `2026-06-26-expo-router-tilt.md`
- 하루에 여러 주제를 다룰 경우 파일을 나눠도 되고, 한 파일에 섹션으로 묶어도 됩니다.
- "무엇을 했는지"보다 **"무엇을 알게 됐고, 왜 그렇게 했는지"**를 남기는 데 집중합니다.

## 템플릿

새 로그를 작성할 때 아래 템플릿을 복사해서 사용하세요.

```markdown
# YYYY-MM-DD · 주제

## 배경 / 상황
<!-- 어떤 작업을 하다가, 어떤 문제를 만났는지 -->

## 알게 된 것
<!-- 새로 배운 개념, 동작 원리, 라이브러리 특성 등 -->

## 시도 / 해결 과정
<!-- 어떤 접근을 시도했고, 무엇이 동작/실패했는지 -->

## 결론
<!-- 최종적으로 어떻게 처리했는지, 왜 그 방법을 택했는지 -->

## 참고
<!-- 관련 링크, 문서, 커밋, PR 등 -->
```
