# Draw Destiny — 설계 문서

> **컨셉:** "운명을 뽑아라". 질문을 고르고, 카드 덱을 기울이거나 화살표로 스크롤해
> 운명 카드를 뽑는 모바일 게임. React Native + Rive 학습용 프로젝트.

**작성일:** 2026-06-25

---

## 1. 목표와 범위

### MVP 목표

사용자가 질문을 선택하고, 셔플된 카드 덱을 기울이기/화살표로 스크롤하며,
가운데 카드를 탭해 플립으로 운명 텍스트를 확인하고, 뽑은 카드를 컬렉션에 모은다.

### 학습 포인트

- **Rive**: 랜딩 연출, 덱 shuffle, 카드 reveal 상태 머신 (Community 무료 에셋 활용)
- **React Native**: expo-router 화면 전환, 탭 네비게이션, 상태 관리, AsyncStorage
- **센서**: accelerometer로 기울임 입력
- **제스처/애니메이션**: Reanimated 4 + Gesture Handler로 캐러셀 스크롤, 카드 플립
- **연동**: 센서 값 → 카드 스크롤 → Rive input 변경

### MVP 제외 (고도화 단계)

- **자유 질문 입력**: 사용자가 직접 질문과 카드 텍스트/개수를 입력하는 기능
- 흔들기(shake)로 덱 섞기 — MVP는 "섞기" 버튼으로 대체

## 2. 아키텍처

### 화면 구조 (expo-router)

```
src/app/
  _layout.tsx          # 루트 스택 (헤더 없음)
  index.tsx            # ① 랜딩 — 탭 없음, 로고 + 애니메이션 + "게임 시작" 버튼
  (tabs)/
    _layout.tsx        # 탭 네비게이터 (여기서부터 탭 등장)
    game.tsx           # ② 게임 — 질문 선택 → 덱 스크롤 → 뽑기/플립
    collection.tsx     # ③ 컬렉션 — 도감
```

- 랜딩(`index`)은 루트 스택의 독립 화면 → 탭바 없음
- "게임 시작" → `router.replace('/(tabs)/game')` (뒤로가기 시 랜딩 안 보이게)
- 게임/컬렉션은 탭바로 전환

### 기술 스택

| 영역       | 라이브러리                                              | 비고                                                   |
| ---------- | ------------------------------------------------------- | ------------------------------------------------------ |
| 라우팅     | expo-router ~56                                         | 기설치                                                 |
| 애니메이션 | react-native-reanimated 4, react-native-gesture-handler | 기설치                                                 |
| Rive       | `rive-react-native`                                     | **신규** — 코드 작성 전 v56 호환 버전 확인             |
| 센서       | `expo-sensors`                                          | **신규** — accelerometer                               |
| 저장       | `@react-native-async-storage/async-storage`             | **신규**                                               |
| 콘텐츠     | `src/data/seed-decks.json`                              | 초기 시드 데이터 (고도화 시 서버/사용자 입력으로 대체) |

> **주의:** 신규 의존성 추가 전 https://docs.expo.dev/versions/v56.0.0/ 에서
> Expo SDK 56 호환 버전을 반드시 확인할 것.

## 3. 데이터 모델

### 질문 + 카드 세트 (`src/data/seed-decks.json`)

각 항목이 "질문 1개 + 그 질문 전용 카드 덱" 한 세트(=deck)다.
`seed-` 접두사는 고도화 단계에서 서버/사용자 입력으로 대체될 임시 초기 데이터임을 나타낸다.

```json
[
  {
    "id": "caution-today",
    "question": "오늘 조심해야 할 것은?",
    "cards": [
      { "id": "c1", "text": "말보다 행동을 앞세우지 마세요", "symbol": "🌙" },
      { "id": "c2", "text": "서두르면 놓칩니다", "symbol": "🐢" },
      { "id": "c3", "text": "오늘의 직감을 믿으세요", "symbol": "✨" }
    ]
  }
]
```

**규칙:**

- 각 덱(질문)은 **전용 카드 세트**를 가진다 — 코드에서 `deck.question`, `deck.cards`로 접근.
- 카드 세트는 **질문당 최소 8장** — 보이는 창(5장) + 순환 스크롤이 자연스럽도록.
- MVP는 드롭다운 선택 질문만 (자유 입력은 고도화 단계).

### 컬렉션 저장 (AsyncStorage)

```
키: "collection"
값: ["c1", "c3", ...]   // 뽑아본 카드 id 목록 (중복 제거)
```

읽기 실패 시 빈 배열로 시작 (크래시 방지).

## 4. 화면별 상세

### ① 랜딩 (`index.tsx`)

- 탭바 없음.
- 중앙에 "Draw Destiny" 로고 + 약간의 Rive 애니메이션 연출.
- 하단 "게임 시작" 버튼 → `router.replace('/(tabs)/game')`.

### ② 게임 (`(tabs)/game.tsx`)

진입 흐름: **질문 선택 → 덱 등장 → 스크롤 → 뽑기/플립**

**1단계 — 질문 선택**

- 화면 상단 드롭다운에서 질문 선택.
- 선택 시 해당 질문의 카드 세트를 셔플하여 덱 구성.

**2단계 — 덱(캐러셀)**

- 전체 셔플 덱 위를 **5장 창(window)**이 스크롤하는 컨베이어 방식.
- 모든 카드는 **뒷면(문양)** 으로 표시.
- **가운데(3번째) 카드가 활성** — 살짝 위로 떠오르고 확대, 그림자 강조.
- **스크롤은 순환(무한 루프)** — 덱 끝에서 처음으로 이어져 항상 양옆에 카드가 있음.

```
전체 셔플 덱: [B F D Z C G A ...]
보이는 창(5):  [B F D Z C]   (가운데 D 활성)
오른쪽 기울임 →  [G B F D Z]   (왼쪽에서 G 진입, 오른쪽 C 이탈)
```

**스크롤 입력 (두 방식 모두 항상 제공)**

- **기울이기**: `expo-sensors` accelerometer x값 → 스무딩(저역통과) + 임계값 + 디바운스 → 한 칸씩 이동.
- **좌우 화살표 버튼**: 항상 표시. 책상에 두고 플레이하는 경우 대비. 기울이기와 동일한 "한 칸 스크롤" 동작 호출.

**3단계 — 뽑기 + 플립**

- 활성(가운데) 카드 탭 → 카드가 중앙으로 이동/확대.
- 제자리 플립: `rotateY` 0°→180°, 90°에서 앞/뒷면 교체.
- 플립 완료 시 Rive `reveal` 트리거 → 운명 텍스트 + 심볼 등장.
- 뽑은 카드 id를 AsyncStorage 컬렉션에 저장 (중복 무시).
- 뽑힌 동안 센서/스크롤 정지. "다시 뽑기"로 해제.

**섞기 버튼**

- Rive `shuffle` 애니메이션 + 카드 배열 셔플.
- 셔플 후 창 위치 리셋, 뽑힘 상태 초기화.

### ③ 컬렉션 (`(tabs)/collection.tsx`)

- 전체 카드를 그리드로 표시.
- 뽑아본 카드 → 앞면(컬러, 텍스트).
- 아직 안 뽑은 카드 → 잠김(회색 뒷면 + 자물쇠).

## 5. 핵심 상태 (게임 화면)

Reanimated shared values 중심:

```
selectedPromptId : 선택된 질문
deck             : 셔플된 카드 배열
windowStart      : 보이는 창의 시작 인덱스 (순환)
scrollOffset     : 스크롤 애니메이션 진행 값 (shared value)
isDrawn          : 카드 뽑힘 여부
flipProgress     : 플립 진행도 (0=뒷면, 1=앞면)
```

데이터 흐름:

```
accelerometer x / 화살표 탭 → scrollStep(direction) → windowStart 갱신 → withSpring 캐러셀 이동
가운데 카드 탭 → isDrawn=true → flip 애니메이션 → Rive reveal → AsyncStorage 저장
```

## 6. 엣지 케이스 & 기술 처리

- **센서 미지원/권한 없음**: 화살표 버튼으로 스크롤 항상 가능 (기울이기는 부가 입력).
- **기울임 떨림**: x값 스무딩 + 임계값 + 디바운스 → 한 칸씩만 이동.
- **Rive 로드 실패**: 정적 카드 뒷면 이미지로 폴백.
- **AsyncStorage 읽기 실패**: 빈 컬렉션으로 시작.
- **뽑은 상태 잠금**: 카드 뽑힌 동안 센서/스크롤 정지, "다시 뽑기"로 해제.

---

## 7. 단계별 구현 순서 (권장)

1. **라우팅 골격**: 랜딩 → 탭(게임/컬렉션) 화면 전환.
2. **데이터 + 질문 선택**: `seed-decks.json`, 드롭다운, 셔플.
3. **캐러셀 + 화살표 스크롤**: 순환 창, 활성 카드 강조 (센서 없이 먼저).
4. **기울이기 입력**: accelerometer 연동.
5. **뽑기 + 플립**: Reanimated 플립, AsyncStorage 저장.
6. **컬렉션 화면**: 도감 그리드.
7. **Rive 연동**: 랜딩 연출 → shuffle → reveal 순으로 점진 적용.

> 1~6은 Rive 없이 동작 가능 → Rive는 시각 레이어로 마지막에 얹는다.
